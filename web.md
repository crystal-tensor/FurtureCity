项目名称：AI 驱动的2100零碳智慧能源城市

整体视觉风格建议采用“未来科技+绿色生态”的混合视觉风格 。使用干净的浅色底板，配合局部的霓虹高亮和绿色生态元素，避免做成过暗的“赛博朋克”风或单调的纯PPT风格 。

一、 3D模型设计：生物启发与海绵城市的可视化
在网站的中央核心区，放置一个可以通过鼠标/手指拖拽旋转的3D城市街区模型。这个模型需要直观体现出“设计理念”。
* 生态底盘与水循环： 模型的底层设计需借鉴“海绵城市”理念，直观展示雨水收集、蓄存与再利用的结构 。
* 仿生建筑群： * 仿植物采光塔： 建筑物外观应强调高效采光与能量转换界面，模拟人工叶片 。
    * 仿白蚁丘通风中心： 设计独特的建筑形态，强调其被动通风与建筑热环境调节功能 。
    * 仿莲叶外墙： 设定建筑表面具有自清洁功能，提升设备维护效率 。
* 动态能量流： 在3D模型上，用不同颜色的动态光带（能量流动画）代表不同的能源流向 。

二、 网站交互设计：让城市“动”起来的AI控制台
围绕中间的3D模型，网站的四周是UI数据仪表盘，重点展示系统目标：有输入、有算法、有输出 。


1. 左侧面板：环境输入与预测层 (Machine Learning)
* 实时环境控制（输入）： 设置几个滑动条或按钮，允许评委（或演示者）手动改变环境状态，如：天气变换（晴天/暴雨）、光照强度、人口负载、水位等 。
* 短期需求预测（输出）： 展示一个动态折线图。说明系统正在使用机器学习（如回归模型或树模型）预测未来短时间的能源需求、峰值负载和天气变化影响 。这向评委证明了项目“为什么要先预测，再调度”的逻辑 。
2. 右侧面板：决策层与仪表盘 (Reinforcement Learning)
这里是展示算法深度的核心区域。建议避免堆叠复杂术语，而是用评委能看懂的“状态-动作-奖励”机制来展示强化学习 。


* AI 调度状态机：
    * 状态 (State)： 实时显示当前天气、能源需求、储能电量、海绵系统蓄水情况等 。
    * 动作 (Action)： 动态图表显示AI当前决定的太阳能、储能、水循环等多种能源的供给分配比例 。
    * 奖励 (Reward)： 核心指标仪表盘，实时滚动显示碳排放量、能源自给率、储能状态等 。
* 远景拓展： 可以在右下角放置一个不起眼的“极端环境模式”按钮。点击后，3D模型背景切换为火星地貌，说明该系统也可迁移到资源受限的极端环境（ISRU），作为远景设想 。
3. 底部控制条：终极对比演示 (Showcase)
最能打动评委的往往是直观的对比 。在底部设置一条类似于视频播放器的控制栏，包含核心演示模式：


* 时间滑块： 设置一个100年时间滑块，拖动时展示城市的演化过程 。
* 一键对比开关 (A/B Test)：
    * 日常运行对比： 开启前，显示能源分配粗放，峰值负载压力大；开启AI优化后，展示预测先行，负载更平滑，整体效率提升 。
    * 极端天气响应： 模拟暴风雨或极寒。开启前，供能不稳定、部分区域断电；开启AI优化后，展示系统如何自动调度储能与备用方案，保持城市稳定运行 。

三、 评审现场答辩策略
这段演示将把一份静态的概念图，变成一个“会思考、会响应、可被展示”的动态城市系统 。


讲解话术建议：
“各位评委老师好。本项目借鉴了中国海绵城市建设与美国智慧城市的数字孪生经验 。大家现在看到的是我们通过前端技术搭建的仿真界面。当遇到极端暴雨天气时（拖动天气滑块），大家可以看到机器学习算法已经提前预测了峰值负载 。接着，强化学习网络开始介入，通过动态调配仿生建筑收集的能源与储能系统 ，在保证城市不拉闸限电的同时，让碳排放指针始终保持在零碳区间 。这不仅是未来的地球城市，其内核算法同样适用于未来的火星拓荒 。”
